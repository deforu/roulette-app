# Webルーレット 現行実装・開発状況

最終更新日：2026年7月12日  
対象バージョン：`roulette-app-v29`  
メインファイル：`index.html`

---

## 1. プロジェクト概要

ブラウザ上で動作する、ジャンル分類・重み付け・履歴管理に対応したルーレットアプリです。

候補をジャンルごとに登録し、ジャンルの表示・非表示や当選比率を設定できます。抽選結果は「決定」または「保留」で処理し、決定済み候補は履歴として保持します。

現在はHTML、CSS、JavaScriptを1つの `index.html` にまとめた構成です。外部フレームワークやバックエンドは使用していません。

---

## 2. ファイル構成

```text
roulette-app-v29/
├── index.html
├── ドキュメント.md
├── roulette-spin.mp3      # 利用者が配置する回転音
└── roulette-result.wav    # 利用者が配置する結果音
```

音声ファイルは必須ではありません。配置されていない場合でも、ルーレット本体は動作します。

---

## 3. 起動方法

`index.html` をブラウザで開くと起動します。

基本機能はローカルファイルとして開いた場合でも動作します。ただし、画像のクリップボードコピーや共有機能は、ブラウザによってHTTPSなどのセキュアコンテキストが必要です。

開発時はローカルHTTPサーバー経由で開く方が、ブラウザAPIの動作確認をしやすくなります。

## 5. 候補とジャンルの登録

### 5.1 編集単位

「現在のジャンル」の各ジャンルカードが編集単位です。カード右側の鉛筆ボタンを押すと、候補一覧を開いた状態で次の入力欄へ切り替わります。

- ジャンル：任意、1つの文字列
- 候補：改行区切りで複数入力
- 比率
- 表示・非表示

例：

```text
ジャンル：食べ物
候補：
寿司
カレー
ラーメン
```

この場合、3件の候補がすべて「食べ物」ジャンルに属します。

### 5.2 編集操作

- 鉛筆：既存ジャンルを編集
- `✓`：編集内容を保存してルーレットへ即時反映
- `×`：編集内容を破棄
- ごみ箱：確認後にジャンルと現在の候補を削除
- 「＋ ジャンルを追加」：一覧末尾に新規編集カードを追加

編集は同時に1ジャンルだけ行えます。編集中はルーレット回転、他ジャンルの設定変更、CSV操作、初期化を無効化します。候補が1件もない場合や比率が不正な場合は保存できません。

### 5.3 同名ジャンルの扱い

現在は、同じジャンル名は同一ジャンルとして統合されます。

編集後のジャンル名が既存ジャンルと同名になった場合は、候補を重複排除して既存ジャンルへ統合します。統合先の以下の設定を維持します。

- 表示・非表示
- ジャンル比率
- 現在のジャンル一覧上の表示

ジャンル未入力の候補は、すべて「ジャンルなし」として1つにまとめます。

### 5.4 候補の重複

候補入力欄内では、同じ文字列の候補は重複排除されます。

選択済み履歴の重複判定は、次の組み合わせで行います。

```text
ジャンル名 + 候補名
```

同じ候補名でもジャンルが異なる場合は、別の候補として扱います。

## 6. ジャンルフィルター

ジャンルごとに表示・非表示を切り替えられます。

### 表示中

- ルーレットの抽選対象に含まれる
- 残り候補数と全体候補数の計算対象に含まれる

### 非表示

- 入力データ自体は削除されない
- ルーレットには表示されない
- 抽選対象から除外される
- 残り候補数と全体候補数の計算から除外される

### 一括操作

- すべて表示
- すべて非表示

PCのジャンル欄と、スマートフォン・タブレットのドロワーの両方から操作できます。

## 7. 比率設定

### 7.1 比率ON

ジャンル全体の当選確率を、ジャンルごとの比率で決定します。

例：

```text
食べ物：比率1
ゲーム：比率2
```

この場合、ジャンル全体の当選割合は次のようになります。

```text
食べ物：1 / 3
ゲーム：2 / 3
```

ジャンル内に複数候補がある場合、そのジャンルへ割り当てられた確率を候補数で均等に分割します。

比率の初期値は `1`、最小値は `0.1` です。

### 7.2 比率OFF

ジャンル単位の重み付けを使用しません。

表示中の全候補を、ジャンルに関係なく同じ確率で抽選します。

比率OFF中は各ジャンルの比率入力欄を操作できません。ただし、入力済みの比率値は保持され、再度ONにすると復元されます。

## 8. ルーレットの回転

### 8.1 回転開始

「ルーレットを回す」を押すと、ルーレットへ回転力を加えます。

現在は固定時間・固定停止位置方式ではなく、次の値を持つ速度ベースの回転処理です。

- 現在角度
- 角速度
- クリック時の加速量
- 摩擦・減速率
- 最大速度
- 停止判定速度

### 8.2 追加クリック

回転中もボタンを押せます。

- ボタン表示は「さらに回す」に変化
- 押すたびに回転力を追加
- 追加加速量にはランダム差がある
- 最大速度を超えない
- 押し続ける限り回転を継続できる

### 8.3 現在の設定値

```text
初回加速：22 ～ 36
追加加速：5.5 ～ 16
最大角速度：42 rad/s
停止判定：0.18 rad/s以下
```

最大速度は約6.7回転/秒です。

### 8.4 回転中の結果表示

回転開始直後から、固定された針が現在指している候補を選択結果欄へリアルタイム表示します。

- ジャンルあり：`ジャンル：候補`
- ジャンルなし：`候補`

停止時に針が指している候補が最終結果になります。

---

## 9. 決定・保留

### 回転前

- 「ルーレットを回す」を表示
- 「決定」「保留」は非表示

### 回転中

- 「ルーレットを回す」または「さらに回す」を表示
- 「決定」「保留」は非表示

### 停止後

- 「ルーレットを回す」は非表示
- 「決定」「保留」を表示

### 決定

- 候補入力欄から対象候補を削除
- ルーレット候補から削除
- 選択済み一覧へ追加
- すでに同じ「ジャンル + 候補」が選択済みの場合は選択回数を加算

### 保留

- 候補を削除しない
- 選択済み一覧にも追加しない
- 次の抽選へ戻る

## 10. 選択済み一覧

選択済み候補をジャンルごとに分類して表示します。

例：

```text
食べ物
- カレー（3回）
- 寿司

ゲーム
- マインクラフト（2回）
```

候補が1回だけ選択されている場合は、回数を表示しません。

タイトル横の件数は、重複を含む選択回数の合計です。

---

## 11. 復活機能

### 11.1 個別復活

各選択済み候補の右側に「復活」ボタンがあります。

復活すると、次の処理を行います。

- 選択済み一覧の記録は残す
- 同じジャンルの候補入力欄の末尾へ追加
- 再びルーレットで抽選可能にする

同じジャンルの入力欄が存在しない場合は、新しい入力グループを末尾へ追加します。

### 11.2 復活済み表示

選択済み候補が現在のルーレット候補にも存在する場合は、次の状態になります。

- 選択済み一覧の文字色を薄く表示
- 個別の復活ボタンを無効化
- 重複して候補入力欄へ追加しない

### 11.3 全復活

選択済み一覧の右下に「全復活」ボタンがあります。

- 未復活の候補を一括で候補入力欄へ戻す
- すでに復活中の候補は追加しない
- 選択済み一覧と選択回数は保持する

---

## 12. 候補数の表示

ルーレットの針の上に次の形式で表示します。

```text
残り候補数 / 全体候補数
```

### 残り候補数

表示中ジャンルのうち、現在のジャンルカードに登録されている候補数です。

### 全体候補数

表示中ジャンルについて、次の候補を統合して重複を除いた数です。

- 現在の候補
- 選択済み候補

重複判定単位は「ジャンル + 候補名」です。

非表示ジャンルは両方の数から除外されます。

## 13. CSVインポート・エクスポート

### 13.1 用途

CSVは候補登録だけでなく、ルーレット設定全体の保存・復元に使用します。

### 13.2 CSV列

```csv
ジャンル,候補,ジャンル比率,ジャンル表示,選択済み,選択回数,比率設定
```

| 列 | 必須 | 空欄時の扱い |
|---|---:|---|
| 候補 | 必須 | エラー |
| ジャンル | 任意 | ジャンルなし |
| ジャンル比率 | 任意 | 1 |
| ジャンル表示 | 任意 | true |
| 選択済み | 任意 | false |
| 選択回数 | 任意 | 選択済みなら1 |
| 比率設定 | 任意 | true |

### 13.3 真偽値として利用できる値

例：

```text
true / false
1 / 0
yes / no
on / off
表示 / 非表示
選択済み / 未選択
有効 / 無効
```

### 13.4 インポート

- 現在の設定をCSV内容で置き換える
- 同名ジャンルを統合
- 比率を復元
- 表示・非表示を復元
- 選択済み状態と選択回数を復元
- 比率ON / OFFを復元

必須列「候補」がない場合や、不正な比率・選択回数がある場合はエラーを表示します。

### 13.5 エクスポート

次の内容をすべて出力します。

- 現在の候補
- 選択済み候補
- ジャンル
- ジャンル比率
- 表示状態
- 選択済み状態
- 選択回数
- 比率ON / OFF

UTF-8 BOM付きCSVとして出力するため、Excelで日本語が文字化けしにくい形式です。

ファイル名の形式：

```text
roulette-settings-YYYYMMDD-HHMM.csv
```

## 14. サウンド機能

### 14.1 必要なファイル

```text
roulette-spin.mp3
roulette-result.wav
```

`index.html` と同じフォルダに配置します。

### 14.2 回転音

- 回転開始時に `roulette-spin.mp3` を1回再生
- ループしない
- 音声が先に終了しても再生し直さない
- ルーレット停止時に停止して先頭へ戻す
- 追加クリックでは再生し直さない

### 14.3 結果音

- 最終結果が表示された瞬間に `roulette-result.wav` を1回再生
- ループしない

### 14.4 サウンドON / OFF

ヘッダー右側のスピーカーアイコンから切り替えます。

OFFにすると、回転音と結果音の両方を再生しません。回転中にOFFへ切り替えた場合は、再生中の音声を停止します。

## 15. 選択済み一覧の画像出力

選択済み一覧の下部に次のボタンがあります。

- 保存
- コピー
- 共有

右側には「全復活」を表示します。

### 15.1 出力画像

PNG形式の縦長画像をCanvasで生成します。

画像には次の内容を含めます。

- タイトル
- 出力日時
- 全候補数
- ジャンル名
- 候補名
- 重複回数

全候補数は次の形式です。

```text
重複を含む選択回数 / 重複を除いた候補数
```

例：

```text
全候補数：8 / 5
```

重複を除く単位は「ジャンル + 候補名」です。

### 15.2 保存

PNGファイルをダウンロードします。

ファイル名の形式：

```text
selected-items-YYYYMMDD-HHMMSS.png
```

### 15.3 コピー

Clipboard APIに対応している場合は、PNG画像をクリップボードへコピーします。

未対応の場合は、PNG保存へフォールバックします。

### 15.4 共有

Web Share APIのファイル共有に対応している場合は、端末の共有シートを開きます。

未対応の場合は、PNG保存へフォールバックします。

### 15.5 ブラウザ上の制約

画像コピーやファイル共有は、ブラウザ・OS・HTTPS環境によって利用可否が異なります。

- コピー：`ClipboardItem` と `navigator.clipboard.write` が必要
- 共有：`navigator.share` とファイル共有対応が必要
- ローカルファイルで開いた場合、コピーが拒否される場合がある
- iOS、Safari、一部Androidブラウザでは挙動差がある

## 16. 内部データ構造

### 16.1 inputGroups

保存済みのジャンルと候補入力文字列を保持します。画面上では同名ジャンルを1枚のカードとして扱い、編集保存時にも1グループへ正規化します。

```text
id
ジャンル
候補入力文字列
```

### 16.2 genreEditor

編集中の一時データを保持します。保存するまで `inputGroups` とルーレット候補は変更しません。

```text
新規 / 既存
編集前のフィルターキー
ジャンル名
候補入力文字列
比率
表示状態
編集元画面（PC / ドロワー）
```

### 16.3 allCandidates

入力欄から構築した全候補です。

```text
入力グループID
フィルターキー
ジャンル
候補名
```

### 16.4 candidates

表示中ジャンルに属する候補だけを保持します。実際のルーレット描画・抽選に使用します。

### 16.5 filterSettings

ジャンル単位の設定を保持するMapです。

```text
表示状態
比率
```

ジャンル名から生成したキーで管理するため、同名ジャンルは同じ設定を共有します。

### 16.6 selectedItems

選択済み履歴を保持します。

```text
ジャンル
候補名
選択回数
保存時の比率
保存時の表示状態
```

## 19. 既知の制約・確認事項

### 19.1 HTMLタイトルのバージョン表記

`index.html` の `<title>` は現在、次の旧表記が残っています。

```text
追加入力で加速できるルーレット v25
```

実ファイルは `roulette-app-v29` です。次回修正時にタイトル表記を更新する必要があります。

### 19.2 データの永続保存

入力内容と次の設定は、操作のたびにブラウザの `localStorage` へ自動保存されます。

- 入力グループ、ジャンル、候補
- ジャンルの表示状態と比率
- 選択済み履歴と選択回数
- 比率設定のON/OFF
- サウンド設定

保存内容は次回のページ読み込み時に自動復元されます。ブラウザのサイトデータを削除した場合や、`localStorage` を利用できない閲覧環境では保持されません。CSVエクスポート・インポートは、ファイルとしてのバックアップや別端末への移行に使用できます。

### 19.3 画像コピー・共有

コードは実装済みですが、ブラウザAPIの対応状況に依存します。非対応時は画像保存へフォールバックします。

### 19.4 Canvas画像の日本語フォント

画像生成にはOS・ブラウザの標準 `sans-serif` を使用します。端末によって字形が若干異なります。

### 19.5 非表示ジャンルと選択済み一覧

非表示ジャンルの履歴自体は選択済み一覧に残ります。ただし、ルーレット上部の残り候補数・全体候補数には含めません。

### 19.6 候補の識別

候補の同一性は文字列の完全一致とジャンル名で判定します。表記揺れ、前後の意味的な同義語、全角・半角差を自動統合する処理はありません。

## 21. 開発時の注意

現在はすべてのCSSとJavaScriptが `index.html` に含まれています。

機能追加が続く場合は、保守性のため次のように分割することを推奨します。

```text
index.html
styles.css
js/
├── state.js
├── roulette.js
├── candidates.js
├── csv.js
├── image-export.js
├── sound.js
└── ui.js
```

状態管理、描画、CSV、画像出力、サウンドが1ファイルに集中しているため、今後の変更では既存機能への影響確認が必要です。
